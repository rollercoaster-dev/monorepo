# Harness Engineering

Adapt harness engineering principles for agent-first development. Restructure docs as system of record, add mechanical architecture enforcement, make the application legible to agents, and build continuous hygiene processes.

## Phases

### Phase 0: Foundation

Documentation infrastructure — index, plans reorganization, CLAUDE.md audit. Establishes the system of record that all subsequent phases depend on.

### Phase 1: Architecture Enforcement

Mechanical checks that prevent architectural drift. Dependency rules, import boundaries, and automated validation.

### Phase 2: Legibility

Make the codebase readable by agents: consistent naming, clear module boundaries, discoverable entry points.

### Phase 3: Continuous Hygiene

Automated processes that maintain quality over time: stale doc detection, dependency audits, coverage tracking.
