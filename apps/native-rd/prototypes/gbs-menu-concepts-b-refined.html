Concept B — Refined

Three variations on the floating-dock idea, each pulling from a different design tradition.

Refining: the original Concept B had three weak spots — (1) the orphan Settings corner button felt visually unbalanced, (2) the + FAB was ambiguous (new goal? new badge?), (3) tab/FAB/tab proportions were uneven. Each variation below fixes a different combination of these issues.
Variation B1

Focus Pill

Borrowed from iOS Focus-mode indicators: only the active tab gets a labeled pill — inactive tabs collapse to icon-only circles. Settings sits as the trailing dot inside the same pill (no orphan corner button). The dock breathes.

  • Lightest visual footprint of the three
  • Active context is unmistakable — text + color cue
  • Asymmetric weight = active tab feels "in focus"
  • FAB stays small/yellow — not the center of gravity

Variation B2

Speed-Dial Dock

Material 3 FAB Menu pattern. Tap + and it expands into 3 contextual creation actions — kills the "what does + do" ambiguity. Settings folds into the dock as a fourth tab. Symmetric, no orphans.

  • Most powerful — surfaces all primary creation paths
  • Tap-to-expand keeps idle UI minimal
  • Dock has 4 tabs (G, B, S + a content tab) — even spacing

Variation B3

Adaptive Dock

The dock changes shape based on the active tab. Goals/Badges = fully expanded with a contextual + FAB ("New Goal" / "New Badge"). Settings = the FAB hides, the dock collapses to a slim 3-tab strip. Inspired by iOS 26's context-aware lock screens.

  • Most ambitious — questions whether chrome should be static
  • Settings tab gets less weight by design (rarely used)
  • FAB label adapts: "Goal" on Goals tab, "Badge" on Badges tab
  • Try it: tap the ⚙ tab to see it collapse
