GBS Bottom Menu — 3 Redesign Concepts

Replacing the single-letter "G / B / S" tab bar in native-rd. Each concept addresses a different problem with the current pattern.

Concept A

Icon + Label Pills

Each tab becomes a discrete brutalist button with an icon and an uppercase label. The active tab pops forward via a shifted hard shadow — readable in every theme without relying on subtle text color shifts.

  • Lowest-risk change — same shape, same 3-tab model
  • Fixes the "what does G mean" affordance gap
  • Active state visible to color-blind & low-vision users (shape + position cue)

Concept B

Floating Dock + Action

The bar floats off the edge as a pill-shaped dock. A centered + button (New Goal) becomes the primary action. Settings demotes to a corner button — it's used rarely and shouldn't share weight with daily actions.

  • Surfaces the most common action: creating a new goal
  • Reduces chrome weight — content gets more vertical space
  • Trade-off: only 2 navigation tabs in the dock; settings moves out

Concept C

Segmented Color Slabs

Three full-bleed colored slabs (purple / yellow / pink) with a thick black indicator that slides under the active tab. Inactive slabs desaturate. Brings more of the brand palette into the chrome instead of one solid purple slab.

  • Most visually distinctive — strongest brutalist statement
  • Active state cued by saturation + indicator (redundant encoding)
  • Risk: heavier visually; needs muting in autismFriendly / lowInfo themes
